WebSocket load test

One mic source, many concurrent transcription sockets.

This page opens a configurable number of WebSocket connections to /ws, reuses one live microphone stream for every socket, and keeps browser memory low with a fixed 5 MB log buffer.

Audio is captured once and streamed to all open sockets. The log view is capped at 5 MB so the browser keeps memory use bounded while the socket pool stays alive.
